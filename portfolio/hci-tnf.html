thenotesrefinery


Collaboration for Version 3.0


15 Sep 2018 – 15 Oct 2018

Introduction

thenotesrefinery is a website that facilitates the collaborative notes sharing process by creating a platform for students to upload their notes, help one another and improve together. Through the websites, users are able to search, upload and share O and A level notes.

About The Project


I am honoured to be able to collaborate with thenotesrefinery to design the preliminary User Interface (UI) for Version 3.0.

2 main reasons for the need to change the current UI:

  • Change in verification process, where registered members of thenotesrefinery are now the ones managing the verification process. Once a note has been verified by 5 members, it will be accessible for all to download.
  • The current UI uses WordPress plugin. However, due to the excessive number of plugins used, it results in slow response of the web pages.

Research

Problems with Current Interface


Search

When users first enter the website, the most basic function (search) is not prominently shown in the home page. The search bar is actually hidden in the side navigation bar, which is not intuitive.



Search List

Users are restricted to only filter the notes by subject type and there is limited flexibility to the search function. Moreover, details of each note can only be viewed upon mouse hover, making it very inefficient for users to navigate through every note. In addition, the details only included the title of the note, without any other descriptions like the author, description, ratings etc.



Login

Violates Nielsen’s Heuristics H-4 (Consistency and standards)

User login is placed at the bottom of the home page but almost every functionality, for example viewing/downloading notes, requires user to login. This placement is unconventional, and most users will face difficulties in finding it.



Navigation Bar

Violates Nielsen’s Heuristics H-6 (Recognition rather than recall)

A side navigation bar is used in the current version. The section ‘Subjects’ is always highlighted in bold despite the user being on different pages. This could be misleading to the users when they are on another section of the website.



Colour Scheme

Violates Nielsen’s Heuristics H-4 (Consistency and standards)

The colour scheme used is inconsistent. In general, it follows a royal blue theme but in the center of the home page, 3 buttons with 3 different colours (blue, green and red) are used, disrupts the theme of the page.

Moreover, there is no correlation between the colours chosen and the functionality of the buttons. For example, why is the ‘Sharing your notes’ in green but ‘Referring 5 friends’ in red? The colours used could mislead the user as green is usually used for positive/agreement while red is used for negative/disagreement.

Design

The following are the solutions to solve the problems identified previously.


Search

As this is a note searching website, I decided to go for a minimalist design approach as the main purpose of the website is for users to navigate around and search for notes easily. Following Nielsen’s Heuristics H-8 (Aesthetic and minimalist design) and analysing minimalist search engines like Google and OneSearch (NLB catalogue search), I decided that the search bar should be the first item on the home page as this is the primary function that users will be using.



Search List

This column layout is chosen for displaying the list of notes as it allows users to view information easily and more details of each note (author, date uploaded, views, ratings etc) are provided in addition to the title.




There is also increase in flexibility to the search function, where users can search for notes using keyword, by the user name, filter by multiple subjects and choosing of level.



Login

Following Nielsen’s Heuristics H-4 (Consistency and standards), the sign in button is placed at the top right corner of the navigation bar, a standard position used by many applications like Google, Facebook and YouTube.



Navigation Bar

The navigation bar is changed to a top navigation bar with a fixed position as it makes it easier for the user to change tabs as it is always accessible to the user. The section on the navigation bar is also correctly colour coded, where the text is darker in colour when the user is on that section, to facilitate the user in recalling which section they are currently in.



Colour Scheme

The colour scheme follows a royal blue theme, similar to the logo of thenotesrefinery.
